# Developer Machine Cleaner

(Internal slug: `dev-cleanup`.) Cross-platform desktop app (Windows / macOS / Linux) that reclaims disk space by
detecting development project stacks and known AI/LLM caches, then moving the
selected artifacts to the OS Recycle Bin / Trash (recoverable).

## Stack

- **Tauri 2** shell — Rust backend, system WebView frontend
- **React 19 + TypeScript 5.8 + Vite 7** for the UI
- **Rust crates:** `walkdir`, `rayon` (parallel sizing), `trash` 5 (cross-platform
  recycle), `dirs` 5 (home/cache/config), `serde`, `anyhow`
- **Tauri plugins:** `tauri-plugin-dialog` (folder picker), `tauri-plugin-opener`

## Commands

```bash
npm run tauri dev      # dev (vite + cargo build, watches Rust)
npm run tauri build    # release bundle
npm run build          # frontend-only typecheck + vite build (fast sanity check)
```

When changing files under `src-tauri/`, Cargo recompiles on next `tauri dev`
launch; a running session won't pick them up. Icons baked at build time, so
icon changes need a restart too.

## Architecture

```
src/                              # React + TS UI
  App.tsx                         # sidebar tabs + content shell
  App.css                         # liquid-glass design system
  lib/
    ipc.ts                        # typed wrappers around invoke() calls
    format.ts                     # formatBytes
  components/
    ProjectsPanel.tsx             # folder scan results
    AiCachesPanel.tsx             # AI/LLM cache scan results

src-tauri/
  src/
    lib.rs                        # Tauri builder + command registration
    commands.rs                   # invoke handlers (wrap blocking work in spawn_blocking)
    cleaner.rs                    # trash::delete wrapper, per-path result
    scanner/
      mod.rs                      # shared dir_size() via walkdir
      stacks.rs                   # STACK_RULES → DetectedProject[]
      ai_caches.rs                # SPECS → AiCacheEntry[]
  icons/                          # bundled icon variants (regenerated by tauri-cli)
    _source/                      # generate.py + 1024px source (not bundled)
  tauri.conf.json                 # bundle config; icon array drives window icon
```

### Data model (Rust ↔ TS)

- `DetectedProject { path, name, stacks: string[], cleanable: CleanableDir[], total_cleanable_bytes }`
- `CleanableDir { path, label, size_bytes }`
- `AiCacheEntry { id, name, path, exists, size_bytes, note }`
- `CleanResult { path, ok, error? }`

Serde renames are camelCase-free — fields are snake_case on the wire, so the TS
interfaces use snake_case too. Don't "fix" that.

### Stack detection

`scanner/stacks.rs` walks the user-picked folder up to `max_depth` (default 6),
matches each directory against `STACK_RULES` (presence of marker files like
`package.json`, `Cargo.toml`, `pyproject.toml`, etc.), and reports cleanable
sub-paths (`node_modules`, `target`, `.venv`, `build`, `bin`, `obj`, …). When a
folder matches multiple stacks (polyglot repo), they're merged into one
`DetectedProject` with the union of cleanables.

`SKIP_DIR_NAMES` is the recursion stop-list — never descend into a build
artifact while searching for projects.

### AI cache detection

`scanner/ai_caches.rs` has a static `SPECS` table of well-known cache locations
(HuggingFace, Ollama, PyTorch, TensorFlow, LM Studio, Jan, GPT4All, Cursor,
Claude Desktop, Claude Code). Each entry resolves via `dirs::*` for cross-OS.

**Conservative on purpose:**
- Cursor entries target only volatile caches (`Cache`, `Code Cache`,
  `CachedData`, `GPUCache`, `logs`), never the extensions/settings dirs.
- Claude Code entries skip `~/.claude/projects/` and any conversation/memory
  state — only shell-snapshots, statsig, todos, ide subdirs.

If adding a new AI tool, follow the same rule: only paths the tool will
recreate on next launch.

### Cleaning

`cleaner::clean_paths` iterates the input, calls `trash::delete(path)` for each,
returns `Vec<CleanResult>` with per-path success/error. The frontend confirms
via `window.confirm` before invoking.

## UI conventions

Both panels share the same vertical structure — keep this consistent:

1. Panel bar (primary action + context text)
2. Filter chips row (`All` + per-stack/per-cache chips with counts)
3. Summary plate (count + total size + Select all/none + Clean button)
4. Content list
5. Empty / missing states

State pattern: `selected: Set<string>` keyed by absolute path; `filter: string |
null` drives `visible` via `useMemo`.

### Styling

`src/App.css` is a single design-token sheet (liquid-glass dark). Key tokens:
`--glass-1..--glass-strong` for translucent surfaces, `--hairline*` for borders,
`--accent` (`#7c9bff`) for primary, `--danger` (`#ff6b7a`). Buttons are pill-
shaped with inner-highlight + outer glow. Backdrop blobs (`body::before`) give
the glass something to refract — don't remove.

## Icon

Source: `src-tauri/icons/_source/icon.png` (1024×1024), generated by
`generate.py` (Pillow). To iterate:

```bash
python src-tauri/icons/_source/generate.py
npx tauri icon src-tauri/icons/_source/icon.png
```

The second command rewrites every platform variant (ico, icns, Square*Logo,
iOS, Android) in place.

## Gotchas

- **`spawn_blocking` is mandatory** for scanner/cleaner calls — walkdir is sync
  and a deep scan would freeze the WebView otherwise.
- **Projects with zero reclaimable bytes are filtered out** in the UI
  (`ProjectsPanel.runScan`); the Rust side returns them anyway, the trim is
  intentional UX.
- **Hot reload doesn't apply to Rust or icons** — restart `tauri dev`.
- **`tauri.conf.json` icon array order matters**: the first PNG is the default
  window icon on Linux; `icon.ico` is what Windows embeds as the exe resource.
